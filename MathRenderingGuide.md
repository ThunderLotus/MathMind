# 小学生口算题 UI 渲染与输出规范

本文档提炼了当前生成器在界面展示和打印输出时，如何“完美呈现”算式、嵌套括号、常规分数以及带分数的核心渲染策略。该内容尤其针对无天然数学公式渲染引擎支持的环境（例如原生纯 Web DOM 或普通 GUI 框架绘制），确保迁移后排版依然美观对齐。

## 1. 分数的排版架构核心 (Flex 布局与边框模拟)

当前项目并未依赖 LaTeX 或 MathML 等重型引擎，而是利用标准的 UI 布局盒模型（Box Model）模拟了分数渲染。这种模式极易移植到 Web (HTML/CSS)、移动端 (Android LinearLayout / iOS StackView) 或桌面端 (WPF / WinForms / JavaFX)。

### 1.1 常规分数（真/假分数）呈现

对于分子为 `n`，分母为 `d` 的分数呈现，最核心的思路是：将其封装在一个**“内联列式弹性盒” (Inline-Flex Column)**中。

*   **容器结构**: 创建一个内联容器，垂直居中对齐（这保证了与其他整数、运算符在同一水平线上对齐）。
*   **分子(首行)**:
    *   处于容器上方。
    *   **横轴(分数线)**：通过为其设置**底部边框 (Bottom Border)** 来作为分数线。
    *   **微调**：可以通过 Y 轴小幅下移（例如 1-2个像素）来使分数线正好处于操作符（如 "+"、"="）的居中十字线上。
*   **分母(次行)**:
    *   处于容器下方。
    *   居中对齐，与分子等宽。
    *   可以通过 Y 轴小幅上移，使得分子分母之间的行距最紧凑。

**伪代码/标准 CSS 示例**：
```css
/* 外层容器：与普通文本能够同一行混排，垂直方向互相居中 */
.frac-container {
    display: inline-flex;
    flex-direction: column;
    align-items: center;
    vertical-align: middle;
    margin: 0 2px;
}
/* 分子带着下划线作为分数线 */
.frac-numerator {
    border-bottom: 1.5px solid currentcolor; 
    width: 100%;
    text-align: center;
    line-height: normal;
    padding: 0 3px;
}
/* 分母在下方 */
.frac-denominator {
    text-align: center;
    line-height: normal;
    padding: 0 3px;
}
```

### 1.2 带分数的渲染

带分数的形态是一部分整数，紧接着连着一个分数。

*   **分离计算**：如果 `格式要求 = mixed` 且分子 `n > 分母 d`，程序首先执行带分数拆封：
    *   整数部分 `W = n / d` (向下取整)
    *   新的分子 `n' = n % d`
*   **组合渲染**：
    *   容器变更为：**水平排列** (Inline-Flex Row/横向排布)。
    *   先输出正常的整数格式文本 `W`。
    *   若余数不为0，再在后面紧随渲染一个小字号（如常规字体的 85%）的常规分数组件（包含 `n'` 和 `d`）。缩小字号能让带分数的结构看起来更标准紧凑，不致于过度占用纵向空间。

### 1.3 负数与符号位处理

无论是整数还是分数，其数值在内部对象（AST 节点）上都会标识正负。
*   如果是纯整数，直接将负号紧跟着数字。
*   如果是分数（不管是常规分数还是带分数），负号应该**提取到整个数字块的最左侧**，并且与中央的分数线（而不是分子）处于同一个垂直对齐线。在我们的布局里，即在 Flex 容器内放置横排的最前列。

## 2. 算式结构与嵌套表现 (递归 AST 渲染)

题目的展现采用了对抽象语法树 (AST) 做 **中序遍历 (In-order Traversal)** 的方式。整个算式界面的渲染是一个递归组件。

对于每个当前节点（树）：
1.  **是否是数字叶子节点**？
    *   如果是，终止递归，调用上方描述的“分数/整数渲染逻辑”。
2.  **如果是运算符节点**：
    *   先判定当前子树是否需要外包裹括号（调用上一文档提到的 `checkNeedsParens` 法则）。
    *   **渲染输出左半边**：若需要括号，输出左括号 `(` -> 递归渲染左子树(`node.left`)。
    *   **渲染中心符**：输出本节点的运算符（ `+` `-` `×` `÷` ），并在左右留出极细微的间隔（空格或者 margin），同时略微向上调整坐标让符号匹配分数的中心。
    *   **渲染右半边**：递归渲染右子树(`node.right`) -> 若需要括号，输出右括号 `)`。

## 3. 跨平台移植指南与打印输出

这种渲染方案使得我们可以抛弃对特定依赖库的使用。

*   **如果迁移到 Java (如使用 Swing/JavaFX)**：
    可以自己编写一个复合控件（继承自 JPanel / Pane），利用 BoxLayout / VBox 即可将分子、分母、横线组装，重写 `paintComponent` 甚至直接 `Graphics2D` drawString 画分数的横线是最稳妥且精确的方法。
*   **如果迁移到 Mobile (Android / iOS)**：
    可以使用 `LinearLayout` (垂直, 权重包裹) 结合 `Divider` 或者 `View` 充当分数线。外层加一个 `LinearLayout` (水平) 组装算式。
*   **导出为 Word 或 PDF 文档**：
    通常单纯的 HTML div + border 难以被完美转换为原生 Word 公式。如果你的后端需要直接导出真正的 `.docx`：
    *   建议将生成的算式结构转化为 **MathML** 标准字符串。
    *   或者通过类似 `Apache POI` 配合 `OMML` (Office Math Markup Language) 协议，将 AST 强行转化为原生 Office Equation 元素，这样老师在 Word 里可以直接无缝编辑最终题目。
